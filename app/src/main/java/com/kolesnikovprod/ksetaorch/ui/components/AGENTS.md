# AGENTS.md: `ui/components`

## Что лежит в директории

`ui/components` содержит небольшие переиспользуемые Compose-компоненты с
пиксельной графикой проекта:

- `GradientIcon.kt` окрашивает drawable градиентом;
- `PixelFrames.kt` рисует квадратную и широкую рамки на Canvas;
- `PixelToggle.kt` рисует трек и кнопку переключателя.

Компоненты уже выполняют свою задачу. Трогать их без необходимости не нужно.
Изменения уместны, когда новый экран требует другой параметр, обнаружен
визуальный дефект, меняются исходные drawable-ресурсы или дизайн-система.

## Граница ответственности

Эти файлы отвечают за отрисовку. Они не должны управлять состоянием экрана,
вызывать `ViewModel`, запускать навигацию или выполнять бизнес-логику.

Ожидаемый поток:

```text
Screen получает state и callbacks
    -> передаёт готовые параметры в ui/components
        -> компонент рисует заданное состояние
```

`PixelToggleIcon` служит визуальным индикатором. Родитель обрабатывает нажатие,
меняет state и решает, доступно ли действие.

`PixelSquareFrame` и `PixelWideFrame` рисуют фон с контуром. Они не размещают
контент внутри себя, поэтому вызывающий код собирает слои через `Box` или другой
контейнер.

`GradientIcon` использует белую маску `Icon`, offscreen-композицию и
`BlendMode.SrcAtop`. Удаление любого из этих шагов меняет результат наложения
градиента.

## Правила изменений

- Сначала найти все места использования изменяемого компонента.
- Сохранять stateless API: состояние приходит через параметры.
- Передавать размер и расположение через `Modifier`.
- Не добавлять обработку клика внутрь чисто визуального компонента без задачи
  на изменение его контракта.
- Сохранять `contentDescription` для интерактивных и смысловых иконок.
- Для декоративной иконки разрешать `contentDescription = null`.
- Не включать в эти файлы `ViewModel`, навигацию, install/inference-контуры или
  экранный state.
- Не выносить Canvas-геометрию в новые абстракции только ради сокращения строк.
  Здесь явные координаты помогают сверять пиксельный рисунок.

## Осторожно с пиксельной геометрией

Canvas-функции используют фиксированную сетку в `dp`. Размеры сторон зависят от
размера Canvas, поэтому слишком маленький `Modifier.size` даст отрицательные
размеры прямоугольников:

- `PixelSquareFrame` требует минимум `16.dp` по каждой оси;
- `PixelWideFrame` требует минимум `24.dp` по каждой оси.

`PixelToggleKnob` вырезает фиксированный участок bitmap: `13 x 12` пикселей с
позиции `(25, 26)`. При замене `tumbutton_ok` или `tumbutton_no` нужно проверить
координаты crop и визуальный результат. `FilterQuality.None` сохраняет резкие
пиксельные края.

## Проверка после правок

После изменения:

1. проверить компиляцию Compose-кода;
2. открыть экраны, где компонент используется;
3. сравнить оба состояния toggle;
4. проверить рамки на реальных размерах;
5. проверить accessibility-описание смысловых иконок.

Короткое правило:

```text
ui/components получает готовые параметры и рисует их.
Экран и ViewModel управляют состоянием и действиями.
```
